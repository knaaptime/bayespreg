    "> **Note — lognormal default (since v0.2)**.  `generate_panel_flow_data` \n",
    "now returns strictly-positive flows by default, drawn from $y_t = \\exp(\\eta_t)$ \n",
    "where $\\eta_t = A^{-1}(X_t\\beta + \\alpha + \\varepsilon_t)$ is the latent \n",
    "SAR-filtered linear predictor (also exposed as ``data['eta']``).  The \n",
    "Gaussian-likelihood panel models (`SARFlowPanel`, `SARFlowSeparablePanel`) \n",
    "operate on the latent scale, so we fit on `np.log(data['y'])`.  Pass \n",
    "`distribution=\"normal\"` to recover the legacy Gaussian-on-y behaviour.  \n",
    "The Poisson-likelihood models are unchanged (their DGPs always emit non-negative counts)."

    "## `fit_approx` pattern\n",
    "\n",
    "Each model uses the same basic workflow:\n",
    "\n",
    "1. Construct the panel model.\n",
    "2. Call `fit_approx(...)` with either `method='advi'` or `method='fullrank_advi'`.\n",
    "3. Inspect `model.summary(...)` or `model.inference_data`.\n",
    "\n",
    "In these examples, `n` is the number of optimization iterations used by `pm.fit`, while `draws` is the number of posterior samples drawn from the fitted variational approximation."
   ]
